### Lessons from Building Claude Code How We Use Skills
https://x.com/trq212/status/2033949937936085378

Skills 的一些原则。
1. 不要写太多，重点写非公开知识、可能陷入问题的 bad case 等等
2. 避免明显的共识还要写约束，早年那种分步骤写的 Prompt 现在可能会反而限制 Agent 的适应能力
3. Skills 的描述写的是何时触发，而不是是什么
4. 可以直接把配置写进 Skills 的 config.json 或者让模型调用 AskUserQuestion 的 tool
